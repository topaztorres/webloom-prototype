Webloom

A human-curated web discovery app. Pull a random thread from a shared pool of hand-submitted links, and follow it wherever it leads.

Open the live prototype → View on GitHub

This page is a static project overview. The prototype itself runs an Express + SQLite backend and lives at the link above — it cannot be hosted on GitHub Pages directly.

What it does today

  • Pull a thread. One primary action on the homepage: random site selection with visit tracking.
  • Field Notes. Expandable panel surfacing live metrics — sites, visits, recent additions.
  • Webloom Index. Two views of the same library — a visual Thread Map of categories, and a database-style shelf with cards, tags, and contributors.
  • Submit. Validated link submission with category, tags, and a contributor handle.
  • SQLite-backed API. Express endpoints for listing, random pulls, visit increments, and stats.

How it's built

Frontend

React 18, Vite, TypeScript, Tailwind CSS, shadcn/ui, Framer Motion, Wouter, TanStack Query.

Backend

Express 5 on Node, written in TypeScript and run with tsx in dev, bundled with esbuild for production.

Data

SQLite via better-sqlite3, schema and validators defined once in shared/ using Drizzle ORM and Zod.

Where it's headed

  1. Moderation queue — submissions stop auto-approving.
  2. Contributor pages at /by/:handle.
  3. Curated "threads" as a second axis alongside categories.
  4. Local visit history with a "give me something new" guarantee.
  5. A real hosting story with persistent storage.
  6. A test harness (Vitest) for schema and API smoke tests.

A fuller list lives in the README.
